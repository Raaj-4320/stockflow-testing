## Target architecture rule

Every numeric business field should be one of:

1. **Canonical event data** — immutable source records such as transactions, purchase orders, payments, expenses, stock movements.
2. **Canonical selector output** — live derived output from one approved domain engine.
3. **Snapshot cache** — denormalized cache with owner, sync version, rebuild path, and never used as silent fallback.
4. **Debug/export-only snapshot** — clearly labeled as cached/stored.

No UI, report, export, dashboard, PDF, or accounting flow should display a snapshot as live truth unless it is explicitly marked as a cache/debug value and accompanied by the canonical source status.
